# GPO Assessment Tool

A PowerShell script for the assessment of Group Policy Objects (GPOs) in an Active Directory domain. It produces a consolidated HTML report with findings classified by severity and exports the native (GPMC) report for each GPO.

- **File:** `GPO_Assessment_Tool.ps1`
- **Version:** `1.0`
- **Output language:** Portuguese (pt-BR)
- **Nature:** Read-only with respect to GPOs, AD, and SYSVOL; it writes only to the output directory.

## Requirements

| Requirement | Required | If absent |
|---|---|---|
| Run as Administrator (`#Requires -RunAsAdministrator`) | Yes | The script does not start (blocked by PowerShell). |
| Domain-joined machine | Yes | Terminates during domain detection (`exit 1`). |
| `GroupPolicy` module (RSAT/GPMC) | Yes | Terminates during module import (`exit 1`). |
| `ActiveDirectory` module (RSAT) | No | OU links, the A16 analysis, and cross-domain OU checks are incomplete or empty. |
| Read access to SYSVOL (`\\<domain>\SYSVOL\<domain>\Policies`) | Yes, for A10/A11 | The orphaned GPO analyses are not performed. |
| Read access to GPO delegation (`Get-GPPermission`) | Yes, for A9 | The unresolved SID analysis is incomplete. |
| Recommended credentials | Domain Admin | Partial collection with lower privileges. |

**Fact:** the modules are loaded via `Import-Module`. RSAT can be installed with `Add-WindowsCapability -Online -Name 'Rsat.GroupPolicy.Management.Tools~~~~0.0.1.0'` and `Add-WindowsCapability -Online -Name 'Rsat.ActiveDirectory.DS-LDS.Tools~~~~0.0.1.0'`.

## Execution

```powershell name=README-run.ps1
# Elevated PowerShell, on a domain-joined machine
.\GPO.ps1
```

There are no input parameters. The target domain is detected automatically by:

```powershell name=README-domain-detection.ps1
[System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()
```

## Output

All output is written to `C:\GPO-Assessment\`:

| Item | Path |
|---|---|
| Consolidated report | `C:\GPO-Assessment\GPO-Assessment-Report_<yyyyMMdd_HHmmss>.html` |
| Native (GPMC) report per GPO | `C:\GPO-Assessment\GPO-Exports\<GPOName>.html` |

The consolidated report is written in UTF-8. The individual reports are generated by `Get-GPOReport -ReportType Html`.

## Analyses performed

| Ref. | Analysis | Detection |
|---|---|---|
| A1 | Individual export | Native HTML report of each GPO. |
| A2 | Link map | Links per GPO (Domain, OU, Site) with Enabled/Enforced status, via `Get-GPInheritance`. |
| A3 | Empty GPOs | No `ExtensionData` in Computer and User (based on the XML report). |
| A4 | GPOs without a link (with configuration) | They have configuration but no link. |
| A5 | User only (Computer enabled) | Configurations only in User, with the Computer node still enabled. |
| A6 | Computer only (User enabled) | Configurations only in Computer, with the User node still enabled. |
| A7 | All settings disabled | `GpoStatus = AllSettingsDisabled`. |
| A8 | WMI filters | GPOs with an associated WMI filter. |
| A9 | Unresolved SIDs | Delegation entries whose trustee resolves only to a SID. |
| A10 | Orphaned GPC without GPT | GPC object in `CN=Policies,CN=System` without a corresponding folder in SYSVOL. |
| A11 | Orphaned GPT without GPC | GUID folder in SYSVOL without a corresponding GPC object in AD. |
| A12 | Cross-domain links | Links pointing to a domain different from the GPO's source domain. |
| A13 | Enforced links | Links with Enforced (No Override) active. |
| A15 | Disabled links | Links with `Link Enabled = No`. |
| A16 | GPCO members | Members of `Group Policy Creator Owners` (recursive), via the `ActiveDirectory` module. |

The A1-A16 references preserve the script's internal numbering (`A14` was merged into `A12`; there are no independent `A0` or `A14` sections).

## Behavior and scope

### Facts (code behavior)

- The script is read-only with respect to the environment: it uses only `Get-*` cmdlets and exports; the only write operation occurs in `C:\GPO-Assessment\`.
- It does not run `Set-GPO`, `Set-GPPermission`, or `Remove-GPO`, and it does not modify SYSVOL/AD.
- The GPO collection scope is the current domain (`Get-GPO -All -Domain <domain>`).
- It does not enumerate GPOs from other domains in the forest. Sites are read at the forest scope.
- `$ErrorActionPreference = "SilentlyContinue"` is set globally.
- Failures in individual collections are suppressed and may result in incomplete data with no visible error in the console.

### Inferences (classification and heuristics, not platform facts)

- The severity classification (risk/integrity, hygiene/precedence, inventory) is an editorial convention of this report for prioritization.
- It does **not** correspond to any formal severity defined by Microsoft.
- The detection of cross-domain OU links (A12) uses a regular-expression comparison against the Distinguished Name.
- This is a heuristic and may produce false positives/negatives in topologies with similar domain names.

## Out of scope (recommended complements)

Version `1.0` does **not** assess the items below, which require separate handling:

- Adequacy of security filtering after MS16-072 (presence of `Authenticated Users` / `Domain Computers` with read permission).  
  Ref.: Microsoft, *Can't apply user Group Policy settings if computer objects don't have GPO Read permissions*.
- Presence of credentials in Group Policy Preferences (`cpassword` in XML files in SYSVOL).  
  Ref.: Microsoft Security Bulletin `MS14-025`.
- Content comparison against security baselines (Microsoft Security Compliance Toolkit, CIS Benchmarks, DISA STIG).  
  Ref.: Microsoft Security Compliance Toolkit.
- Configuration-level analysis (individual settings), the ADMX Central Store, and version consistency between AD/SYSVOL.
